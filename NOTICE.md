## Upstream reference: flam3

The fractal-flame algorithm itself is the work of Scott Draves & Erik Reckase. pyr3 inherits
GPL-3.0-or-later from the upstream
[flam3](https://github.com/scottdraves/flam3) C reference renderer (also GPL-3.0-or-later).
The pyr3 engine code is an *independent reimplementation* — we read the C source for
algorithmic clarity but the TypeScript and WGSL in this repo is original code.

## Ported data: flam3 palette library

`src/flam3-palettes-data.ts` (generated by `scripts/gen-flam3-palettes.mjs`) is a **verbatim
port** of flam3's `flam3-palettes.xml` — the canonical 701-palette × 256-RGB library used as
the fallback when a `.flame` references a library palette via `<flame palette="N">` (PYR3-022).
This data originates with **Scott Draves & Erik Reckase** and is redistributed here under
**GPL-3.0-or-later**, consistent with the upstream flam3 license. Only the byte values are
ported (base64-packed for size); no flam3 *code* is copied.

## Reference fixtures (Electric Sheep `.flame` genomes)

Fixture flames are drawn from the Electric Sheep distributed-rendering corpus
([electricsheep.org](https://electricsheep.org), Draves et al., 2004–). Electric Sheep
submissions carry one of two Creative Commons licenses (**CC-BY 2.0** or **CC-BY-NC 2.0**)
depending on the contributor — note that the **CC-BY-NC** genomes constrain commercial use,
including how pyr3.app may be monetized. Per-fixture attribution + license is tracked in
[`fixtures/flam3-goldens/README.md`](fixtures/flam3-goldens/README.md).

## Third-party npm dependencies

Enumerated in `package.json`. The only dependency *distributed to end users* (bundled into the
deployed viewer) is `brotli-dec-wasm`; the rest are dev/build-time only.

- **[`brotli-dec-wasm`](https://www.npmjs.com/package/brotli-dec-wasm)** (MIT OR Apache-2.0,
  both GPL-compatible) — wasm brotli decoder, code-split + fetched only on Chromium (which
  lacks native `DecompressionStream("brotli")`) to inflate corpus chunks (`src/brotli.ts`).
  **This is the one runtime-distributed dependency.**
- **[`webgpu`](https://www.npmjs.com/package/webgpu)** (BSD-3-Clause) — Google Dawn team's
  Node.js bindings for WebGPU. Provides the CLI consumer's `navigator.gpu` shim.
- **[`vite`](https://vitejs.dev)** (MIT) — frontend build tool + dev server.
- **[`vitest`](https://vitest.dev)** (MIT) — test runner.
- **[`tsx`](https://www.npmjs.com/package/tsx)** (MIT) — TypeScript execution for Node.
- **[`pngjs`](https://www.npmjs.com/package/pngjs)** (MIT) — PNG encoding for the CLI output
  path.
- **[`happy-dom`](https://www.npmjs.com/package/happy-dom)** (MIT) — `DOMParser` shim for
  parsing `.flame` XML in the Node CLI.
- **[`@webgpu/types`](https://www.npmjs.com/package/@webgpu/types)** (BSD-3-Clause) —
  TypeScript type definitions for the WebGPU spec.
