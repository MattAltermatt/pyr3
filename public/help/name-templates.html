Flame name templates

The flame's name supports {placeholders} that resolve at save time. The literal template is stored in the genome (so re-opening the file shows the editable text); the filename gets the resolved form.

How {index} works #

Each unique template string keeps its own counter in browser storage. Saving with template sierpinski-chaos-{index} writes sierpinski-chaos-0001.png the first time, then bumps to 0002, 0003, and so on. Switching to a different template ({nick}-{index}) gets a fresh counter starting at 0001; switching back picks up where the old one left off.

The counter advances on every successful save — Save Flame and Save Render each count. A failed save doesn't burn a number. Counters survive page reloads and stay until you clear the browser's localStorage for the pyr3 origin.

Padding is fixed at 4 digits (0001…9999) to match camera DCIM convention. Past 9999 the natural width kicks in.

Escapes #

To produce a literal { or }, double it: {{ becomes {, }} becomes }. So {{literal}}-{date} resolves to {literal}-20260605.

Unknown placeholders (typos like {palete}) are kept literal in the output rather than dropped silently — safer than producing flame--{date} when you meant flame-{palette}-{date}.

Example templates #

{palette}-{index} → south-sea-bather-0001.png
flame-{date}-{index} → flame-20260605-0001.png
{nick}-{datetime} → matt-20260605-1533.png
{palette}-{vars} → south-sea-bather-linear-bent-fan.png
{nick}-{xforms}xf-{seed} → matt-3xf-a1b2c3d4.png
flame-{datetime}-{random} → flame-20260605-1533-3a2f.png

Filename safety #

After resolving, the result is slugified for filesystem safety: lowercase, non-alphanumeric runs collapse to single dashes, leading and trailing dashes are trimmed. So a palette named South Sea Bather becomes south-sea-bather, and a template that resolves to {vars} like linear.bent.fan becomes linear-bent-fan. Plain literal names (no {placeholders}) go through the same slugifier — same behavior as before templates existed.
