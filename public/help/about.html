pyr3

A WebGPU fractal-flame renderer — for the browser and the command line.

Open an Electric Sheep / flam3 .flame file in a WebGPU-capable browser and watch the chaos-game iteration render directly on your GPU. Corpus sheep have a shareable coordinate URL (/esf/gen/{gen}/id/{id}) you can hand to anyone; a flame opened from disk renders locally and has no auto-shareable link. It all runs in the browser; there's nothing to install.

One engine, two ways in: this browser viewer for looking and sharing, and a headless CLI (npm run render) that renders a flame to a PNG at full quality. Both run the same TypeScript + WebGPU code. A visual editor (/editor) and screensaver (/screensaver) are fully integrated.

The pyr3 family

  • pyr3 This renderer — one TypeScript + WebGPU engine with two consumers: the browser viewer (this site) and a headless CLI.
  • ESF The Electric Sheep Fold — a corpus of every Electric Sheep flame, archived and re-collated.
  • flam3 The algorithmic ancestor — Scott Draves's original fractal flame renderer, started in 1992. The chaos-game inner loop + variation library every flam3-family renderer inherits.

Render contract

pyr3 is a single-precision (f32) WebGPU renderer. Like every f32 flame renderer in the family, its output is similar but not pixel-identical to flam3, the canonical f64 reference. Where a genome trips f32-divergent behavior, pyr3 renders its f32 result and moves on.
