WebGPU & pyr3

Why this viewer needs WebGPU, and what to do if your browser doesn't have it yet.

What is WebGPU? #

WebGPU is a web standard that lets a page talk to your graphics card directly. It's the modern successor to WebGL, designed around how GPUs actually work today rather than around the OpenGL ES 2.0 API of 2007.

Fractal flames are computed by running millions of chaos-game iterations per render. That's brutal on a CPU and trivial on a GPU. pyr3 uses five WebGPU compute shaders (chaos · density · spatial filter · two visualize passes) to run those iterations directly on your machine's graphics hardware. No JavaScript loops with that kind of workload would feel acceptable.

If you opened pyr3 and saw a flame rendering, your browser already supports WebGPU. The green WebGPU ✓ chip in the top bar links here just in case you wanted to know what it meant.

Why isn't pyr3 working? #

You're on this section because the chip in pyr3 is red and the canvas didn't load a flame. Your browser isn't exposing navigator.gpu, so pyr3 has no engine to render with.

Browsers that work today:

  • Chrome / Edge 113+ on Windows, macOS, ChromeOS, Android.
  • Safari 18+ on macOS Sequoia (14.5+).
  • Firefox Nightly (behind the dom.webgpu.enabled flag in about:config).

If you're already on one of those, the next likely culprit is a GPU driver that's too old or a hardware-acceleration flag turned off. Chrome's chrome://gpu page will tell you whether WebGPU is enabled and, if not, why.

Is this the only way?

For the browser, yes — and that's deliberate. pyr3 is meant to be the lowest-friction way to share a fractal flame: a URL someone opens in a browser, no install, no native app. WebGPU is the only path that lets a browser do this with anything approaching reasonable performance — pyr3 is GPU-only, so a WebGPU-capable browser is required.
